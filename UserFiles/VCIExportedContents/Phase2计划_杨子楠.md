# Phase 2 周计划

**最近更新:** 2026-05-23 — Module D / E / F PLC 侧全部 VERIFIED；§3 六个模块中 4 个已完成（§3.3 参数化随 §3.4 配方一并完成）、2 个待 NX-MCD 联仿（§3.1 §3.2 的物理验证 V2–V9）。详见各小节状态标记。

## 目标 ( 只做 6 件)

1. **传送带 + 纸盒源联合仿真** — 纸盒由 MCD 生成源生成、随传送带移动、传感器检测到位；HMI 启动后驱动带速并触发生箱
2. **精细码垛逻辑** — SCARA 在传送带末端真实抓取（吸盘握手）→ 码放 → 单箱 6 阶段状态机 → 满箱自动收尾
3. **用配方设定箱体尺寸** — 把箱体尺寸 / 层数 / 速度做成配方，码垛按选定的配方计算
4. **示教功能** — 手动 jog 到目标点 → 一键捕获写入点表；码垛 / ABCDE 路径可选用示教点
5. **双托盘切换** — Pallet 1 码满后自动切换到 Pallet 2；两托盘全满后收尾
6. **参数化 FB** — 把关键 FB 里写死的数值改成可调参数

**不做** (推迟 Phase 3+): 下载到真实硬件 / 多工位多机器人 / pallet 满料物理清盘 / 垛型 pattern 编辑
**保留不动**: Phase 1 的 ABCDE 5 点循环 / HMI 状态显示模块 / 报警 / IO
**不用任何的库**: 延续 Phase 1，只用 MC_MoveLinearAbsolute 等 TO 内置标准 FB

## 阶段总结

主要目标是把 Phase 1 跑通的 ABCDE 单臂循环，升级为一套 **完整、可配置、可示教** 的码垛工作站联合仿真：纸盒由 MCD 产生源真实生成、随传送带到位，SCARA 用吸盘真实抓放，单箱 6 阶段精细控制；箱体尺寸按配方设定，路径点支持示教录入，托盘满后双托盘自动切换，关键 FB 参数化以便复用。

Phase 2 按四步推进：**定决策（步骤 1）→ 搭 NX 仿真环境（步骤 2）→ 逐模块开发、部署、验收（步骤 3）→ 端到端联合仿真验收收尾（步骤 4）**。逐模块开发的 6 个模块依次进行；示教功能在已完成的手动控制功能上扩展，可在精细码垛、配方、双托盘等模块的调试间隙穿插推进。每个模块完成编译下载、功能测试通过后，再进入下一个，不要堆积。
