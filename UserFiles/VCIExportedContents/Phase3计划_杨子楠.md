# Phase 3 周计划

**最近更新:** 2026-05-23 — Phase 2 §3.4 §3.5 §3.6 Module D / E / F PLC 侧全部 VERIFIED（D 14/14 + E V3.0 27/27 + F V1.2 24/24）；Phase 3 开工前提：Phase 2 §4（NX-MCD 联仿 Module G）收尾 + scara-HMI 完成 Phase 2 三块画面（配方 PSC / 双盘按钮 / 示教点表）+ D1/D2 决策定型。**当前状态：未开工**。

## 目标 ( 只做 4 件)

1. **手动垛型编辑** — 在 Web 运行时里拖拽摆箱，编辑每层 pattern；移植之前版本现成的 Web 拖拽编辑器
2. **自动垛型生成** — HMI 输入箱 / 垛参数 → 自写轻量排样算法生成垛型
3. **逐层编辑** — 手动 / 自动生成的垛型，每一层 pattern 可在 HMI 上单独查看、修改
4. **垛型存取** — 编辑完的垛型存入 HMI 存储介质（PSC 或 LJDM），可重新加载

**不做** (推迟以后): 多机型垛型库 / 在线垛型寻优 / 垛型 3D 预览
**保留不动**: Phase 1 + Phase 2 的全部成果（ABCDE 循环、码垛、配方、双托盘、示教等）
**不用任何的库**: 自写排样算法，不用 LPallPatt 等第三方库（延续项目铁律）

## 阶段总结

main target 是给 Phase 2 的码垛工作站补上 **垛型可编辑** 能力：每一层箱体的排布（pattern）既能在 Web 上手动拖拽摆放，也能输入参数由算法自动生成，再逐层精修，最后存进 HMI 存储介质反复调用。

手动侧移植之前版本现成的 Web 拖拽编辑器（成熟的 TypeScript 编辑器）；自动侧自写一套轻量排样算法（不用 LPallPatt —— 延续项目「不用第三方库」铁律）。按步骤 1→5 顺序推进；每个模块完成部署、功能测试通过后再进入下一个，模块一完成就提交 git。

工时估算：约 4 周（~20 个工作日）—— 手动编辑器移植自之前版本的现成模块，自动排样算法为新写。V1–V8 验收点全部逐项核对通过，本阶段收尾。
